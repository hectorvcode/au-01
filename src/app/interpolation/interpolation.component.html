Template Expressions

A template expression produces a velue and appears within double curly braces { { } } Angular resolvers the expression and assigns it to a property of a binding target. The target could be an HTML element, component or directive

Resolving expressions with interpolation

The text between the braces is a template expression that Angular first evaluates and then converts to a string. The following example illustrates this:

The sum or 1 + 1 is { { 1 + 1 } }

The sum of 1 + 1 is {{ 1 + 1 }}

With interpolation, Angular performs the following tasks:
  1. Evaluates all expressions in double curly braces
  2. Converts the expression results to strings
  3. Links the results to any adjacent literal strings
  4. Assigns the composite to an element or directive property

Interpolation is one-way binding

Interpolation is one way as values go from the component to the template, but if you try to change values in the view, the component is not updated

Should not change the state of the app

The template expression should not change the state of the app. Angular uses it to read the values from the compoenent and populate the view. If the template change component values, the rendered view will be inconsistent with the model.

It means you can't make use of the following
  • Assignments(=,+=,-=,...)
  • Keywords line new, typeof, instanceof, etc
  • Chaining expressions with ; or ,
  • The increment and decrement operators ++ and --
  • bitwise operatiors such as | and &

The expression must result in a string

If we return an object it will not work. If you want to bind the expression that is other than a string, then property binding is the best option

Works only on properties and not attributes

This can set only properties, not attributes. For Atttributes you should use attribute binding
